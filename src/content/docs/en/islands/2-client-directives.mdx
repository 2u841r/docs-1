Wen you visit any product page in your online store, you will see the product name, its price, and its description.

These product pages are generated dynamically by `src/pages/producs/[id].astro` and display product information from your `src/data/products.json` file.

![A product page for a wool cap, showing a product name, image, description and cost.](/assets/islands-2-product-page-no-add-to-cart.png)

But, you can't *do* anything on these pages . . . yet! 

Islands allow you to add interactive components in the framework of your choice.

1. Go to `src/pages/products/[id].astro` in your editor and notice which files are being imported: an Astro layout and a JSON data file.

2. Import and add the `<AddToCart />` component as shown below:

```astro title="src/pages/products/[id].astro" ins={4, 28-35}
---
import Layout from "../../layouts/Layout.astro";
import products from "../../data/products.json";
import { AddToCart } from "../../components/AddToCart";

export function getStaticPaths() {
  const pages = products.map((product) => ({
    params: { id: product.id },
    props: { product },
  }));

  return pages;
}

const { product } = Astro.props;
---

<Layout title={product.title}>
  <article class="flex flex-col sm:flex-row sm:gap-8">
    <img
      class="w-full mb-4 sm:mb-0 rounded-md"
      src={product.image}
      alt={product.title}
    />
    <div class="flex flex-col gap-4 w-full">
      <h1 class="text-xl font-semibold sm:text-3xl">{product.title}</h1>
      <p>$ {product.price.toFixed(2)}</p>
      <AddToCart
        product={{
          id: product.id,
          name: product.title,
          price: product.price,
          image: product.image,
        }}
      />
      <p>{product.description}</p>
    </div>
  </article>
</Layout>
```

3. Visit any product page and try adding an item to your shopping cart.

![The same product page for a wool cap, but now also showing a counter to add or subtract quantities, and an add to cart button.](/assets/islands-2-product-page-including-add-to-cart.png)

4. Hydrate the component by adding the `client:load` client directive.

```astro title="src/pages/products/[id].astro" ins={29}
---
import Layout from "../../layouts/Layout.astro";
import products from "../../data/products.json";
import { AddToCart } from "../../components/AddToCart";

export function getStaticPaths() {
  const pages = products.map((product) => ({
    params: { id: product.id },
    props: { product },
  }));

  return pages;
}

const { product } = Astro.props;
---

<Layout title={product.title}>
  <article class="flex flex-col sm:flex-row sm:gap-8">
    <img
      class="w-full mb-4 sm:mb-0 rounded-md"
      src={product.image}
      alt={product.title}
    />
    <div class="flex flex-col gap-4 w-full">
      <h1 class="text-xl font-semibold sm:text-3xl">{product.title}</h1>
      <p>$ {product.price.toFixed(2)}</p>
      <AddToCart
       client:load
        product={{
          id: product.id,
          name: product.title,
          price: product.price,
          image: product.image,
        }}
      />
      <p>{product.description}</p>
    </div>
  </article>
</Layout>
```

5. Visit a product page again and try adding the product to your shopping cart.

### Analyze the Pattern

Given the following `.astro` script:

```astro title="src/pages/about.astro"
---
const operatingSystem = "Linux";
const quantity = 3;
const footwear = "boots";
const student = false;
---
```


For each Astro template expression, can you predict the HTML (if any!) that will be sent to the browser? Click to reveal if you're right!

1.  `<p>{operatingSystem}</p>`

    <p>
      <Spoiler>`<p>Linux</p>`</Spoiler>
    </p>

2.  `{student && <p>I am still in school.</p>}`

    <p>
      <Spoiler>Nothing will display because `student` evaluates to false.</Spoiler>
    </p>

3.  `<p>I have {quantity + 8} pairs of {footwear}</p>`

    <p>
      <Spoiler>`<p>I have 11 pairs of boots</p>`</Spoiler>
    </p>

    </p>
